# 문제 작성 가이드

## 원칙

- 상업용 문제집·유료 해설 복사 금지
- 실제 기출 원문을 연도·회차와 함께 복제하지 않음
- `기출유형` / `출제경향` 표현 + `sourceType: 자체작성_기출유형`
- 숫자·문장만 바꾼 중복 문제 금지

## 유형 비율 (참고)

- 개념 확인 25% · 공식 적용 30% · 계산 30% · 응용 15%

## 필수 필드

고유 ID, 과목, 단원, 유형, 난이도, 중요도, 지문, 보기 4개, 정답 인덱스,
상세 해설, 단계별 계산(계산형), 보기별 오답 이유, 관련 공식·강의,
예상 풀이시간, 자체제작 여부, 검토 상태

## 좋은 해설

1. 사용할 개념·공식부터 제시
2. 단위 변환 → 대입 → 결과
3. 오답 보기가 틀린 이유
4. 빠른 풀이 팁(있으면)
5. 역산·차원 검산

## 정답 분포

정답이 특정 번호에 몰리지 않게 분산. `dart run tool/validate_content.dart`로 확인.

## 계산문제 검증

입력 충분성, 단위, 공식, 중간·최종 계산, 반올림, 정답 유일성,
해설 일치, 관련 공식 링크, 물리적 타당성.
